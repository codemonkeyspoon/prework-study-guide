Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta tag contains information about the webpage like character set and initial viewport size.
  • Rule of thumb is only one h1 tag per webpage, use h2 and so on for subsequent use.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS can be inline, external, or declared within the html, however this inline and declaration can over-boat code which leads external being the most common method.
  • CSS is extremely flexible allowing developers to style webpages to be viewed across PC and mobile sized monitors.
  • Colors in CSS can be chosen by name or an equal hex value.

Git

  • Traditional Git workflow starts with inizializing the repository, then clone to local machine, make changes and commit, push new changes to repository, then merge.
  • It is common practice to create a branch describing the added feature befor merging to the main branch.
  • Clear and concice comments greatly hep during code review.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
